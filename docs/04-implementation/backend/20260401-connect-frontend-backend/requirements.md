# requirements.md — フロントエンド・バックエンド連携

**作成日:** 2026-04-01
**対象ステアリング:** `docs/steering/20260401-connect-frontend-backend/`

---

## 概要

フロントエンドのデータ永続化層を LocalStorage から Backend API（REST）に切り替える。
ユーザーが入力・操作した内容をバックエンド経由で PostgreSQL に保存・取得できるようにする。

---

## ユーザーストーリー

- ユーザーとして、タスクを追加したとき、DBに保存されることを期待する
- ユーザーとして、タスクの完了状態を変更したとき、DBに反映されることを期待する
- ユーザーとして、タスクを削除したとき、DBから削除されることを期待する
- ユーザーとして、完了済みタスクをまとめて削除したとき、DBから削除されることを期待する
- ユーザーとして、API呼び出し中はローディング状態が視覚的に分かることを期待する

---

## 受け入れ条件

- [ ] タスクの追加・完了切替・削除・一括削除がすべてAPIを通じてDBに反映される
- [ ] ページリロード後もタスク一覧がDBから復元される（LocalStorageに依存しない）
- [ ] API呼び出し中はローディングインジケーターが表示される
- [ ] API呼び出し失敗時はコンソールにエラーが出力される（今回はUIへの表示は最小限）
- [ ] `docker compose up` で全サービスが起動し、End-to-Endで動作する

---

## 制約事項

- バックエンドAPIのエンドポイント・DTOは変更しない
- フロントエンドの見た目（Tailwind CSSによるスタイル）は極力変えない
- TypeScript の型安全性を維持する
- LocalStorage 関連コードは削除する（後方互換不要）
