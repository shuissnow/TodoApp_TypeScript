# requirements.md — 期限日（due_date）機能の追加（バックエンド）

**作成日:** 2026-04-09

---

## 1. 概要

Todo に期限日（due_date）フィールドを追加する。
ユーザーが任意で期限日を設定・解除でき、期限日によるフィルタリングとソートをバックエンドで実現する。

---

## 2. ユーザーストーリー

```
US-DD-BE-01: 期限日を設定してタスクを作成する
  ユーザーとして、
  タスク作成時に期限日を指定したい
  なぜなら、いつまでに完了すべきかを管理できるから

US-DD-BE-02: 既存タスクの期限日を変更・解除する
  ユーザーとして、
  タスクの期限日を後から変更したり、なしに戻したりしたい
  なぜなら、予定が変わることがあるから

US-DD-BE-03: 期限日でタスクを絞り込む
  ユーザーとして、
  期限切れ・今日期限・今週中・期限あり・期限なしでタスクを絞り込みたい
  なぜなら、優先度の高いタスクに集中できるから

US-DD-BE-04: 期限日順にタスクを並べ替える
  ユーザーとして、
  期限日の近い順にタスクを表示したい
  なぜなら、優先順位を把握しやすいから
```

---

## 3. 機能要件

### 3.1 データモデルの変更

| フィールド | 型 | 必須 | 説明 |
|-----------|-----|------|------|
| due_date | date（日付のみ） | ✗ | 期限日（省略可能、時刻なし） |

- `due_date` は既存 Todo モデルに追加するフィールド
- null 許容（期限なしを表す）
- 過去の日付も保存可能（バリデーションはフロントエンドで行う）

### 3.2 API 変更

#### POST `/api/todos`（タスク作成）

リクエストボディに `dueDate` フィールドを追加（任意）:

```json
{
  "text": "タスク名",
  "dueDate": "2026-04-30"  // 省略可
}
```

#### PUT `/api/todos/{id}`（タスク更新）

リクエストボディに `dueDate` と `resetDueDate` を追加:

```json
{
  "dueDate": "2026-04-30",   // 期限日を設定（任意）
  "resetDueDate": true        // true のとき due_date を null にリセット
}
```

> `dueDate` が未指定かつ `resetDueDate` が false の場合は、既存の期限日を変更しない。

#### GET `/api/todos`（タスク一覧取得）

クエリパラメータを追加:

| パラメータ | 値 | 説明 |
|-----------|-----|------|
| `filter` | `today` | 今日が期限のタスク |
| `filter` | `overdue` | 期限切れのタスク（期限日 < 今日） |
| `filter` | `this_week` | 今日から 6 日以内が期限のタスク |
| `filter` | `has_due_date` | 期限日が設定されているタスク |
| `filter` | `no_due_date` | 期限日が設定されていないタスク |
| `sort` | `due_date` | 期限日昇順・期限なしは末尾 |

- `filter` と `sort` は独立して指定可能
- 未指定の場合は現在の動作（全件・作成日順）を維持

### 3.3 レスポンス

全エンドポイントで `dueDate` フィールドをレスポンスに含める:

```json
{
  "id": "...",
  "text": "タスク名",
  "completed": false,
  "createdAt": "2026-04-09T00:00:00Z",
  "dueDate": "2026-04-30"   // null の場合は省略またはnull
}
```

---

## 4. 非機能要件

- 既存のテスト（Controller・Service）が引き続きパスすること
- EF Core マイグレーションで安全にスキーマを変更すること（既存データに影響なし）
- `filter` / `sort` は DB レベルで処理し、メモリ上での絞り込みは行わない

---

## 5. 受け入れ条件

- [ ] `POST /api/todos` で `dueDate` を含めてタスクを作成できる
- [ ] `POST /api/todos` で `dueDate` を省略してもタスクを作成できる
- [ ] `PUT /api/todos/{id}` で `dueDate` を更新できる
- [ ] `PUT /api/todos/{id}` で `resetDueDate: true` を送ると期限日が null になる
- [ ] `PUT /api/todos/{id}` で `dueDate`・`resetDueDate` どちらも未指定のとき、期限日が変わらない
- [ ] `GET /api/todos?filter=today` で今日が期限のタスクのみ返る
- [ ] `GET /api/todos?filter=overdue` で期限切れのタスクのみ返る
- [ ] `GET /api/todos?filter=this_week` で今日から6日以内が期限のタスクのみ返る
- [ ] `GET /api/todos?filter=has_due_date` で期限日ありのタスクのみ返る
- [ ] `GET /api/todos?filter=no_due_date` で期限日なしのタスクのみ返る
- [ ] `GET /api/todos?sort=due_date` で期限日昇順・期限なしは末尾で返る
- [ ] パラメータなしの `GET /api/todos` は従来通り全件・作成日順で返る
- [ ] マイグレーション適用後、既存データの `due_date` は `null` になる

---

## 6. スコープ外

- フロントエンドの UI 実装（別作業単位で実施）
- 過去日付のバリデーション（フロントエンド側で対応）
- 完了ステータスによるフィルタ（既存機能・変更なし）
