# requirements.md — ログイン機能追加（フロントエンド）

## 変更・追加する機能の説明

バックエンドの Cookie 認証に対応するため、フロントエンドにログイン画面と認証状態管理を追加する。
未認証状態ではログイン画面のみを表示し、認証後に既存の Todo 画面・マスタ画面へ遷移できるようにする。

## ユーザーストーリー

- ユーザーとして、ログイン画面でユーザー名とパスワードを入力してログインしたい
- ユーザーとして、ログイン失敗時にエラーメッセージを確認したい
- ユーザーとして、ヘッダーに自分のユーザー名が表示されていることを確認したい
- ユーザーとして、ヘッダーのログアウトボタンを押してログアウトしたい
- ユーザーとして、未認証状態でアプリにアクセスしたとき自動的にログイン画面が表示されてほしい

## 受け入れ条件

- アプリ起動時に `GET /api/auth/me` でセッション確認を行い、未認証なら即座にログイン画面を表示する
- ログイン成功後、Todo 画面に遷移してデータ取得を開始する
- ログイン失敗時、フォーム下部にエラーメッセージ（「ユーザー名またはパスワードが正しくありません」）を表示する
- ヘッダーにログイン中のユーザー名（またはアバター頭文字）を表示する
- ヘッダーにログアウトボタンを設置し、押下後はログイン画面へ戻る
- 全ての `fetch` リクエストに `credentials: 'include'` を付与して Cookie を送受信する

## 制約事項

- ユーザー登録（サインアップ）画面は今回スコープ外
- ハッシュルーティング（`#/...`）の仕組みは変更しない。ログイン画面はハッシュなし（ルートパス相当）で表示する
- ログイン画面のデザインは既存の Atom（TextInput・Button）を再利用する
