# requirements.md — 優先度機能追加（フロントエンド）

## 変更・追加する機能の説明

バックエンドに実装済みの優先度（Priority）マスタをフロントエンドに統合する。
Todo 作成時に優先度を選択でき、一覧表示で優先度バッジを確認・変更できるようにする。

## ユーザーストーリー

- ユーザーとして、Todo 作成時に優先度（高/中/低）を任意で指定したい
- ユーザーとして、テーブルビューで各 Todo の優先度をバッジで確認したい
- ユーザーとして、テーブルビューで優先度バッジをクリックして優先度を変更したい
- ユーザーとして、ボードビューで各 Todo の優先度をバッジで確認したい

## 受け入れ条件

- Todo 作成フォームに優先度セレクトが表示される（「なし」がデフォルト）
- 優先度セレクトは選択中の優先度の色（foregroundColor / backgroundColor）に変化する
- `POST /api/todos` リクエストに priorityId が含まれる（なし選択時は含まない）
- テーブルビューの優先度列に優先度バッジが表示される
- `priority = null` の Todo は「低」バッジとして表示される
- テーブルビューで優先度バッジをクリックするとセレクトに切り替わり、変更できる
- `PUT /api/todos/{id}` リクエストに priorityId が含まれて更新される
- ボードビューのカードに優先度バッジが表示される
- デザインは緑をベースカラーとする

## 制約事項

- 優先度マスタの CRUD UI は今回対象外
- 優先度によるフィルター・ソートは今回対象外
- ボードビューでの優先度変更は今回対象外（表示のみ）
