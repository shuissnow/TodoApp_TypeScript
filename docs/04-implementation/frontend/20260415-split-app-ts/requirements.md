# requirements.md

## 変更・追加する機能

`src/frontend/src/app.ts` を責務ごとに複数ファイルへ分割するリファクタリング。  
外部から観察できる振る舞いは変えない（機能追加なし）。

---

## ユーザーストーリー

- 開発者として、`app.ts` の肥大化を解消し、各ファイルが単一責務を持つ構成にしたい
- 開発者として、新しい画面やアクションを追加する際に、どのファイルを変更すればよいか明確にしたい

---

## 受け入れ条件

- 既存の全機能が変わらず動作すること
  - Todo の作成・完了切り替え・削除・期限日更新・優先度更新
  - 完了済み Todo の一括削除
  - フィルター切り替え（all / active / completed）
  - ビュー切り替え（リスト / ボード）
  - 優先度マスタの名称・色・有効/無効の更新
- `app.ts` のコード量が大幅に削減され、hash ルーティングと初期化のみになっていること
- ファイル間に循環依存が存在しないこと
- 既存の単体テストが全て通ること

---

## 制約事項

- 外部から観察できる振る舞いを変えない（純粋なリファクタリング）
- 新機能・UI変更は行わない
- `MasterTop.ts` / `MasterPriority.ts` / `AppLayout.ts` は変更しない
