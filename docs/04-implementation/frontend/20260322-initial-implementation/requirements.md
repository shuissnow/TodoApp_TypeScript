# 初回実装 要求内容

**作業ディレクトリ:** `.steering/20260322-initial-implementation/`
**作成日:** 2026-03-22
**ステータス:** ドラフト

---

## 1. 概要

Todoアプリのv1.0を初回実装する。フロントエンドのみの構成で、データはローカルストレージに保存する。

---

## 2. 実装する機能

`docs/product-requirements.md` のコア機能（v1.0）をすべて実装する。

| ID | 機能 | 説明 |
|----|------|------|
| F-01 | タスク追加 | テキスト入力でタスクを作成 |
| F-02 | タスク完了トグル | チェックボックスで完了状態を切り替え |
| F-03 | タスク削除 | タスクを削除 |
| F-04 | タスク一覧表示 | 全タスクをリスト表示 |
| F-05 | フィルタリング | すべて / 未完了 / 完了済みで絞り込み |
| F-06 | データ永続化 | ローカルストレージに保存・読み込み |

追加機能（v1.1相当）も合わせて実装する：

| ID | 機能 | 説明 |
|----|------|------|
| F-07 | 残タスク件数表示 | フッターに未完了タスク数を表示 |
| F-08 | 完了済み一括削除 | 完了済みタスクをまとめて削除 |

---

## 3. ユーザーストーリー

```
US-01: タスクを追加する
  ユーザーとして、
  テキストを入力してタスクを追加したい
  なぜなら、やることを忘れずに記録したいから

US-02: タスクを完了にする
  ユーザーとして、
  完了したタスクにチェックを入れたい
  なぜなら、何が終わったかを視覚的に確認したいから

US-03: タスクを削除する
  ユーザーとして、
  不要なタスクを削除したい
  なぜなら、リストをすっきり保ちたいから

US-04: タスクをフィルタリングする
  ユーザーとして、
  未完了のタスクだけを表示したい
  なぜなら、今やるべきことに集中したいから

US-05: データが保持される
  ユーザーとして、
  ページを閉じても再度開いたときにタスクが残っていてほしい
  なぜなら、毎回入力し直すのは手間だから
```

---

## 4. 受け入れ条件

### US-01 タスクを追加する
- [ ] テキストを入力してEnterキーでタスクが追加される
- [ ] テキストを入力して「追加」ボタンをクリックでタスクが追加される
- [ ] 空白のみのテキストではタスクを追加できない
- [ ] タスク追加後、入力フィールドが自動でクリアされる

### US-02 タスクを完了にする
- [ ] チェックボックスをクリックすると完了 / 未完了が切り替わる
- [ ] 完了タスクのテキストに取り消し線が表示される
- [ ] 完了タスクはグレーアウトされる

### US-03 タスクを削除する
- [ ] 削除ボタンをクリックするとタスクが即時削除される

### US-04 タスクをフィルタリングする
- [ ] 「すべて」タブで全タスクが表示される
- [ ] 「未完了」タブで未完了タスクのみ表示される
- [ ] 「完了済み」タブで完了タスクのみ表示される
- [ ] フィルター切り替えは即時反映される
- [ ] 現在のフィルターがハイライト表示される

### US-05 データが保持される
- [ ] ページリロード後もタスクの内容・完了状態が保持される

### 追加機能
- [ ] フッターに未完了タスクの件数が表示される
- [ ] 「完了済みを削除」ボタンで完了タスクが一括削除される
- [ ] 完了済みタスクが0件の場合、「完了済みを削除」ボタンは非表示になる

---

## 5. 制約事項

- バックエンドAPIは使用しない（ローカルストレージのみ）
- 外部UIライブラリは使用しない（Tailwind CSS のみ）
- 認証機能は実装しない
- タスク編集機能は今回のスコープ外
