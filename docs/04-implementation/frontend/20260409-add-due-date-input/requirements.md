# requirements.md — 期限日入力・編集・表示の追加（フロントエンド）

**作成日:** 2026-04-09

---

## 1. 変更・追加する機能の説明

タスクの作成時および編集時に期限日（dueDate）を任意入力・変更できるようにする。
また、タスク一覧に期限日を表示する。
バックエンドはすでに `dueDate` の受け取り・保存・返却・リセット（`resetDueDate`）に対応済み。

---

## 2. ユーザーストーリー

> ユーザーとして、タスクを追加する際に期限日を入力できるので、いつまでに完了すべきかを管理できる。

> ユーザーとして、作成済みのタスクの期限日をインラインで変更・削除できるので、スケジュール変更に対応できる。

---

## 3. 受け入れ条件

### 3.1 タスク作成

- [ ] タスク入力フォームに日付選択 input が表示される
- [ ] 日付は任意入力（空欄でもタスクを追加できる）
- [ ] タスク追加時、入力した日付が API リクエストの `dueDate` に含まれる
- [ ] タスク追加後、テキストと日付の両方がリセットされる

### 3.2 タスク編集（期限日）

- [ ] テーブルビューの「締め切り」列でインライン編集できる（クリックで date input に切り替わる）
- [ ] 日付を変更すると `PUT /api/todos/{id}` で `dueDate` が更新される
- [ ] 日付を空欄にして確定すると `resetDueDate: true` で期限日が削除される
- [ ] 編集をキャンセル（Escape キー）すると変更が破棄される

### 3.3 表示

- [ ] テーブルビューの「締め切り」列に `dueDate` が `YYYY-MM-DD` 形式で表示される
- [ ] 期限日なしのタスクは `—` を表示する
