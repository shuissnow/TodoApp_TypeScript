# 要件定義書 — E2E テスト（Playwright）

**作成日:** 2026-04-16  
**ステータス:** ドラフト

---

## 1. 背景・目的

現在はユニットテスト（Vitest / xUnit）のみ整備されており、ブラウザを介した統合的な動作確認は手動。  
Playwright を使ったブラウザ自動テストを導入し、主要ユーザーフローを自動で検証できるようにする。

---

## 2. スコープ

| 対象 | 内容 |
|------|------|
| テストフレームワーク | Playwright（`@playwright/test`）を frontend プロジェクトに追加 |
| テスト対象フロー | ログイン認証・Todo CRUD 操作 |
| CI 統合 | GitHub Actions E2E ワークフロー（`e2e.yml`）の新規作成 |
| テスト実行環境 | Chromium（ヘッドレス） |

---

## 3. 機能要件

### FR-01: 認証フローのテスト（`e2e/auth.spec.ts`）

- 正しいユーザー名・パスワードでログインすると Todo 画面に遷移する
- 誤ったパスワードでログインするとエラーメッセージが表示される
- 未認証状態で Todo 画面にアクセスするとログイン画面にリダイレクトされる

### FR-02: Todo 操作フローのテスト（`e2e/todo.spec.ts`）

- ログイン後、テキスト入力でタスクを作成できる
- 作成したタスクのチェックボックスをクリックすると完了状態に切り替わる
- タスクの削除ボタンをクリックするとタスクが一覧から消える
- フィルター（全件 / 未完了 / 完了）で表示を切り替えられる

### FR-03: CI での自動実行（`.github/workflows/e2e.yml`）

- main への PR 作成・更新時にトリガーする
- テスト失敗時はスクリーンショット・トレースをアーティファクトとして保存する
- GitHub Checks にテスト結果を表示する

---

## 4. 非機能要件

- E2E テスト全体の実行時間は 5 分以内を目標とする
- テスト用 DB は docker-compose で起動した PostgreSQL を使用する
- テストデータは各テストの `beforeEach` でセットアップ・クリーンアップする

---

## 5. スコープ外

- Firefox / Safari / Edge でのクロスブラウザテスト（Chromium のみ）
- スマートフォン画面サイズでのテスト（デスクトップのみ）
- 優先度マスタ画面のテスト

---

## 6. 前提条件

- ログイン・セッション機能の実装が完了していること
- バックエンド API が起動した状態でテストを実行できること（docker-compose または直接起動）
