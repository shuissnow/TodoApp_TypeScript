# ユビキタス言語定義

**バージョン:** 1.1
**作成日:** 2026-03-22
**ステータス:** ドラフト

---

## 1. ドメイン用語の定義

| 日本語 | 英語 | 定義 |
|--------|------|------|
| タスク | Todo | ユーザーが管理する1件の作業項目。テキスト・完了状態・作成日時を持つ |
| タスク一覧 | Todo List | ユーザーが登録したタスクの集合 |
| 完了 | Completed | タスクが終わった状態。チェックボックスがオンの状態 |
| 未完了 | Active | タスクがまだ終わっていない状態。チェックボックスがオフの状態 |
| フィルター | Filter | 表示するタスクを条件で絞り込む機能 |
| 一括削除 | Delete Completed | 完了済みのタスクをすべて一度に削除する操作 |

---

## 2. UI/UX用語の定義

| 日本語 | 英語 | 定義 |
|--------|------|------|
| 入力フォーム | Todo Input | タスクのテキストを入力するフィールドと追加ボタンの組み合わせ |
| フィルタータブ | Filter Bar | 「すべて」「未完了」「完了済み」を切り替えるタブUI |
| タスクアイテム | Todo Item | 一覧に表示される1件のタスク行（チェックボックス・テキスト・削除ボタン） |
| フッター | Footer | 残タスク件数と一括削除ボタンを表示するエリア |
| 取り消し線 | Strikethrough | 完了タスクのテキストに引かれる線。完了状態を視覚的に示す |

---

## 3. 技術用語の定義

| 日本語 | 英語 | 定義 |
|--------|------|------|
| ローカルストレージ | Local Storage | ブラウザに内蔵されたキーバリュー形式のデータ保存領域 |
| 永続化 | Persistence | データをリロード後も保持すること |
| コンポーネント | Component | UIの構成単位。独立した描画ロジックと表示を持つ |
| 状態 | State | アプリケーションの現在のデータ（タスク一覧・フィルター選択など） |
| バリデーション | Validation | ユーザー入力の妥当性チェック（例: 空文字のタスクを登録させない） |
| ビルド | Build | ソースコードを実行可能な形式に変換する処理（フロントエンド: JS/CSS変換、バックエンド: .NETコンパイル） |
| デプロイ | Deploy | ビルドされた成果物をサーバーに配置して公開する操作 |
| エンティティ | Entity | DBのテーブルに対応するクラス。EF Core が管理する |
| DTO | DTO (Data Transfer Object) | APIのリクエスト・レスポンスに使うデータ転送専用クラス。エンティティを直接公開しないために使用する |
| リポジトリ | Repository | DBアクセスを抽象化する層。`IRepository` インターフェース経由で利用する |
| マイグレーション | Migration | DBスキーマの変更をコードで管理し、順序通りに適用する仕組み（EF Core） |
| 依存性注入 | DI (Dependency Injection) | クラスが依存するオブジェクトを外部から注入する設計パターン。テストのしやすさと疎結合を実現する |

## 4. 英語・日本語対応表（コード上の命名）

### フロントエンド — 型名

| コード上の名前 | 日本語の意味 |
|--------------|-------------|
| `Todo` | タスク（1件） |
| `TodoList` | タスク一覧 |
| `FilterType` | フィルターの種類 |

### フロントエンド — フィルター値

| コード上の値 | 日本語表示 | 意味 |
|------------|-----------|------|
| `'all'` | すべて | 全タスクを表示 |
| `'active'` | 未完了 | 未完了タスクのみ表示 |
| `'completed'` | 完了済み | 完了タスクのみ表示 |

### フロントエンド — 関数名

| コード上の名前 | 日本語の意味 |
|--------------|-------------|
| `addTodo()` | タスクを追加する |
| `toggleTodo()` | タスクの完了状態を切り替える |
| `deleteTodo()` | タスクを削除する |
| `clearCompleted()` | 完了済みタスクを一括削除する |
| `saveTodos()` | タスク一覧をローカルストレージに保存する |
| `loadTodos()` | タスク一覧をローカルストレージから読み込む |

### フロントエンド — プロパティ名（`Todo` 型）

| コード上の名前 | 日本語の意味 | 型 |
|--------------|-------------|-----|
| `id` | 識別子 | `string`（UUID形式） |
| `text` | タスクのテキスト内容 | `string` |
| `completed` | 完了フラグ | `boolean` |
| `createdAt` | 作成日時 | `string`（ISO 8601） |

### バックエンド — クラス名

| コード上の名前 | 日本語の意味 |
|--------------|-------------|
| `Todo` | タスクエンティティ（1件） |
| `TodosController` | タスクAPIコントローラー |
| `ITodoService` / `TodoService` | タスクサービス（インターフェース / 実装） |
| `ITodoRepository` / `TodoRepository` | タスクリポジトリ（インターフェース / 実装） |
| `CreateTodoRequest` | タスク作成リクエストDTO |
| `UpdateTodoRequest` | タスク更新リクエストDTO |
| `AppDbContext` | アプリケーションDBコンテキスト |

### バックエンド — メソッド名（`ITodoService`）

| コード上の名前 | 日本語の意味 |
|--------------|-------------|
| `GetAllAsync()` | 全タスクを取得する |
| `CreateAsync()` | タスクを作成する |
| `UpdateAsync()` | タスクを更新する |
| `DeleteAsync()` | タスクを削除する |
| `DeleteCompletedAsync()` | 完了済みタスクを一括削除する |

### バックエンド — プロパティ名（`Todo` エンティティ）

| コード上の名前 | 日本語の意味 | 型 |
|--------------|-------------|-----|
| `Id` | 識別子 | `Guid`（UUID） |
| `Text` | タスクのテキスト内容 | `string` |
| `Completed` | 完了フラグ | `bool` |
| `CreatedAt` | 作成日時（UTC） | `DateTime` |

## 5. 略語・頭字語

| 略語 | 正式名称 | 意味 |
|------|---------|------|
| SPA | Single Page Application | 単一HTMLで動作するWebアプリ |
| UUID | Universally Unique Identifier | 世界中で一意なID文字列 |
| GUID | Globally Unique Identifier | UUIDの別称。C#では `Guid` 型として使用 |
| API | Application Programming Interface | システム間の接続インターフェース |
| CRUD | Create / Read / Update / Delete | データの基本操作4種 |
| CI/CD | Continuous Integration / Continuous Delivery | 自動ビルド・テスト・デプロイの仕組み |
| ORM | Object-Relational Mapping | オブジェクトとDBテーブルを対応付ける仕組み |
| JWT | JSON Web Token | 認証情報を格納するトークン形式 |
| XSS | Cross-Site Scripting | 悪意あるスクリプトを埋め込む攻撃手法 |
| DTO | Data Transfer Object | データ転送専用のオブジェクト |
| DI | Dependency Injection | 依存性注入 |
