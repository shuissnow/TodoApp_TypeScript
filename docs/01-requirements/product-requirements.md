# プロダクト要求定義書

**バージョン:** 1.4
**作成日:** 2026-03-22
**ステータス:** ドラフト

---

## 1. プロダクトビジョンと目的

### ビジョン
新しい技術を実際に手を動かして習得するための、シンプルなTodoアプリ。技術選定・設計・実装・デプロイまでの一連の開発プロセスを体験できる学習プラットフォーム。

### 目的
- 技術スタックの実践（フロントエンド・バックエンド・インフラ）
- 設計・実装・テスト・デプロイの全工程の体験
- プロダクション品質のコードを書く練習

---

## 2. ターゲットユーザーと課題・ニーズ

### ターゲットユーザー
新しい技術を実践したい開発者自身

### 課題・ニーズ

| 課題 | ニーズ |
|------|--------|
| Webアプリを一人で作成したことがない | 設計から実装・デプロイまで一人で完結する経験を積みたい |
| フロントエンド（TypeScript）を扱ったことがない | TypeScriptを使った実際のWebアプリ開発を経験したい |
| バックエンド（C# Web API）を扱ったことがない | C# Web APIを使ったサーバーサイド開発を経験したい |
| AWSを使ったインフラ構築をしたことがない | AWSを使ったデプロイ・インフラ構築を経験したい |
| Dockerを使ったことがない | Dockerを使ったコンテナ化・ローカル開発環境の構築を経験したい |
| CI/CDパイプラインを構築したことがない | GitHub Actionsを使った自動ビルド・デプロイを経験したい |
| テストを書いたことがない | フロントエンド・バックエンド両方で単体テストを書く習慣を身につけたい |

## 3. 主要な機能一覧

### コア機能（v1.0でリリース）
1. **タスク追加** - テキスト入力でタスクを作成
2. **タスク完了** - チェックボックスで完了状態を切り替え
3. **タスク削除** - 不要なタスクを削除
4. **タスク一覧表示** - 全タスクをリスト表示
5. **フィルタリング** - 全件 / 未完了 / 完了済みで絞り込み
6. **データ永続化** - リロードしてもタスクが消えない（PostgreSQL）
7. **完了済みの一括削除** - 完了済みタスクをまとめて削除
8. **残タスク件数の表示** - 未完了タスクの件数を表示

### 将来機能（学習テーマ）

| テーマ | 内容 |
|--------|------|
| ユーザー認証 | JWT・AWS Cognitoを使った認証・認可の実装 |
| タスク編集機能 | 新しいAPIエンドポイントの追加・フロントエンド対応 |
| パフォーマンス改善 | キャッシュ・クエリ最適化 |
| ロギング・監視 | アプリケーションログの収集・AWS CloudWatchによる監視 |
| エラーハンドリング | フロントエンド・バックエンド両方の統一的なエラー処理 |
| E2Eテスト | Playwrightなどを使ったブラウザ自動テストの導入 |
| セキュリティ強化 | HTTPS対応・CORS設定・レートリミット |
| データバックアップ | RDSの自動バックアップ・リストア手順の整備 |

## 4. 成功の定義

### 機能的成功
- [ ] コア機能（全8機能）がすべて正常に動作する
- [ ] フロントエンド・バックエンド・DBが連携して動作する
- [ ] PC・スマートフォンの両方で正しく表示・操作できる
- [ ] デプロイ後、URLでアクセスできる

### 学習的成功
- [ ] TypeScriptを使ったフロントエンド開発を一人で実装できる
- [ ] C# Web APIを使ったバックエンド開発を一人で実装できる
- [ ] AWSを使ってフロントエンド・バックエンドをデプロイできる
- [ ] Dockerを使ったコンテナ化・ローカル開発環境を構築できる
- [ ] GitHub Actionsを使ったCI/CDパイプラインを構築できる
- [ ] フロントエンド・バックエンド両方で単体テストを書ける

## 7. 非機能要件

### パフォーマンス
- 初期ページロード：3秒以内
- タスク操作のレスポンス：即時（100ms以内）

### 対応環境
- ブラウザ：Chrome、Firefox、Safari、Edge（最新版）
- デバイス：PC、タブレット、スマートフォン
- 画面サイズ：320px 〜 1920px（レスポンシブ対応）

### セキュリティ
- 入力値のXSS対策（HTMLエスケープ）
- ユーザー入力の適切なバリデーション（フロントエンド・バックエンド両方）
- SQLインジェクション対策（EF Core使用、生SQL禁止）

### アクセシビリティ
- キーボード操作対応（Tabキーナビゲーション、Enterキー送信）
- 十分なコントラスト比（WCAG AA準拠）
