# docs/ フォルダ — Claude 向けガイド

このファイルは Claude（AI）が `docs/` フォルダ内で作業するときの判断基準を定めます。
人向けの説明は `docs/README.md` を参照してください。

---

## ドキュメント選択ガイド

「どのフォルダを触るか」の判断基準です。

| ユーザーの依頼 | 触るべきフォルダ |
|---|---|
| 機能要件・プロダクト目標が変わった | `01-requirements` |
| システム構成・データモデル・API 設計が変わった | `02-basic-design` |
| コーディング規約・命名規則・フォルダ構成が変わった | `03-detailed-design` |
| 新しい開発作業を始める | `04-implementation/[frontend|backend]/` に新規フォルダを作成 |
| 将来やりたいことのメモ | `04-implementation/backlog.md` |
| Docker / PostgreSQL などのコマンド手順 | `04-implementation/runbooks/` |

---

## 永続的ドキュメント（01〜03）vs 作業単位ドキュメント（04）

| | 永続的ドキュメント | 作業単位ドキュメント |
|---|---|---|
| **場所** | `01-requirements/` 〜 `03-detailed-design/` | `04-implementation/[frontend\|backend]/[YYYYMMDD]-[title]/` |
| **内容** | 「このアプリは何か」「どう作るか」の定義 | 「今回の作業で何をするか」の記録 |
| **更新タイミング** | 設計・方針が根本的に変わるときのみ | 開発作業のたびに新規フォルダを作成 |
| **変更後の扱い** | 最新状態を維持 | 完了後も削除せず履歴として保持 |
