# Frontend

TypeScript（バニラ）+ Vite + Tailwind CSS で構築したTodoアプリのフロントエンドです。

---

## フォルダ構成

```
src/frontend/
├── index.html              ← HTMLの入口（ページの骨格）
├── package.json            ← 使用するライブラリとコマンド定義
├── tsconfig.json           ← TypeScriptの設定
├── vite.config.ts          ← ビルドツールの設定
├── eslint.config.ts        ← コード品質チェックの設定
├── .prettierrc             ← コード整形の設定
├── public/                 ← そのまま配信する静的ファイル
└── src/                    ← アプリのソースコード
    ├── main.ts             ← アプリの起動点
    ├── app.ts              ← 状態管理とロジックの中枢
    ├── types/
    │   └── todo.ts         ← 型定義
    ├── services/
    │   └── storage.ts      ← データの保存・読込
    ├── components/         ← 画面のパーツ（DOM生成）
    │   ├── TodoInput.ts
    │   ├── FilterBar.ts
    │   ├── TodoList.ts
    │   ├── TodoItem.ts
    │   └── Footer.ts
    ├── styles/
    │   └── main.css        ← グローバルCSS
    └── test/               ← テストコード
        ├── components/
        └── services/
```

---

## 各ファイルの役割

### `index.html` — ページの骨格

ブラウザが最初に読み込むHTMLです。`<div id="app">` という空の箱だけ用意して、あとはTypeScriptがその中身を作ります。

```html
<div id="app"></div>
<script type="module" src="/src/main.ts"></script>
```

---

### `src/main.ts` — アプリの起動点

ブラウザがHTMLを読み終えたタイミングで `render()` を1回呼び出します。

```typescript
document.addEventListener('DOMContentLoaded', () => {
  render()
})
```

---

### `src/types/todo.ts` — 型定義

TypeScript固有のファイルです。アプリ全体で使うデータの「形」を定義します。

```typescript
// Todoオブジェクトが持つべきプロパティを定義
export interface Todo {
  id: string
  text: string
  completed: boolean
  createdAt: string
}

// 3つの文字列しか入れられない型（タイポをコンパイル時に検出）
export type FilterType = 'all' | 'active' | 'completed'
```

---

### `src/services/storage.ts` — LocalStorageへの読み書き

ブラウザのLocalStorageを使ってデータを永続化します。

```typescript
export const loadTodos = (): Todo[] => { ... }   // 読み込み
export const saveTodos = (todos: Todo[]): void => { ... }  // 保存
```

---

### `src/app.ts` — 状態管理とロジックの中枢

アプリの状態（データ）と操作ロジックをまとめた中心ファイルです。

状態はこの2変数だけで管理します。

```typescript
let todos: Todo[] = loadTodos()  // Todoリスト
let filter: FilterType = 'all'   // 現在のフィルター
```

状態が変わるたびに `render()` を呼んでDOM全体を作り直します。

| 関数 | 役割 |
|------|------|
| `addTodo(text)` | Todo追加 |
| `toggleTodo(id)` | 完了・未完了を切り替え |
| `deleteTodo(id)` | 削除 |
| `clearCompleted()` | 完了済みを一括削除 |
| `setFilter(f)` | フィルター変更 |
| `render()` | DOM全体を再構築 |
| `setupEventListeners()` | ボタン等にイベントを登録 |

---

### `src/components/` — 画面パーツのDOM生成

各ファイルは「HTML要素を作って返す関数」を1つだけエクスポートします。クリック時の処理は書かず、イベント登録は `app.ts` の `setupEventListeners()` が一括で担当します。

| ファイル | 関数 | 作るもの |
|----------|------|---------|
| `TodoInput.ts` | `createTodoInput()` | テキスト入力＋追加ボタン |
| `FilterBar.ts` | `createFilterBar(current)` | すべて・未完了・完了済みボタン |
| `TodoList.ts` | `createTodoList(todos)` | リスト全体 `<ul>` |
| `TodoItem.ts` | `createTodoItem(todo)` | 1件の `<li>`（チェック・テキスト・削除ボタン） |
| `Footer.ts` | `createFooter(todos)` | 残件数＋「完了済みを削除」ボタン |

---

### `src/test/` — テストコード

`src/` と同じフォルダ構成でテストファイルを配置します。

```
test/
├── components/
│   ├── TodoItem.test.ts
│   ├── TodoList.test.ts
│   └── ...
└── services/
    └── storage.test.ts
```

---

## データの流れ

```
ブラウザ起動
    ↓
main.ts → render()
    ↓
app.ts: LocalStorageからtodosを読み込み
    ↓
components/: DOM要素を生成して #app に挿入
    ↓
setupEventListeners(): ボタン等にイベントを登録
    ↓
ユーザー操作（追加・完了・削除）
    ↓
app.ts: 状態(todos/filter)を更新 → LocalStorageに保存 → render()（再描画）
```

## 設定ファイル詳細

### `tsconfig.json` — TypeScriptコンパイラの設定

#### 出力先・言語バージョン系

| 設定 | 値 | 意味 |
|------|----|------|
| `target` | `ES2023` | ビルド後のJavaScriptをES2023の文法で出力する |
| `module` | `ESNext` | `import/export` 構文をそのまま使う |
| `lib` | `ES2023, DOM, DOM.Iterable` | 使用できる型定義の宣言。`DOM` を入れることで `document.querySelector` 等が型エラーにならない |
| `types` | `vite/client` | Vite固有の型（`import.meta.env` 等）を認識させる |
| `noEmit` | `true` | TypeScriptは型チェックだけ行い、`.js` ファイルを出力しない。実際のビルドはViteが担当 |

#### モジュール解決系

| 設定 | 値 | 意味 |
|------|----|------|
| `moduleResolution` | `bundler` | Viteのようなバンドラー向けのimport解決ルールを使う |
| `allowImportingTsExtensions` | `true` | `import './foo.ts'` のように `.ts` 拡張子付きでimportできる |
| `verbatimModuleSyntax` | `true` | 型だけのimportは必ず `import type` と書かないとエラーになる |
| `moduleDetection` | `force` | すべてのファイルを「モジュール」として扱う |

#### 型チェック厳格化系

| 設定 | 値 | 意味 |
|------|----|------|
| `strict` | `true` | 厳格モード。`null` チェック漏れ・暗黙的 `any` などを一括でエラーにする |
| `noUnusedLocals` | `true` | 宣言したが使っていないローカル変数をエラーにする |
| `noUnusedParameters` | `true` | 宣言したが使っていない関数引数をエラーにする |
| `noFallthroughCasesInSwitch` | `true` | `switch` 文で `break` 忘れをエラーにする |
| `noUncheckedSideEffectImports` | `true` | 副作用目的のimport（`import './styles.css'`）を型チェック対象にする |

### `vite.config.ts` — ビルドツールの設定

```typescript
/// <reference types="vitest" />
import { defineConfig } from 'vite'
import tailwindcss from '@tailwindcss/vite'

export default defineConfig({
  plugins: [tailwindcss()],
  test: {
    environment: 'happy-dom',
  },
})
```

| 設定 | 意味 |
|------|------|
| `/// <reference types="vitest" />` | `test: { ... }` ブロックの型定義をVitestから読み込む指示。これがないと `test:` の部分で型エラーが出る |
| `plugins: [tailwindcss()]` | Tailwind CSS v4をViteのプラグインとして組み込む。使用したユーティリティクラスだけのCSSを生成する |
| `test.environment: 'happy-dom'` | テスト実行時にブラウザAPIを模倣する。Node.jsには `document` や `window` が存在しないため、DOM操作のテストに必要 |

---

### `eslint.config.ts` — コード品質チェックの設定

```typescript
export default tseslint.config(
  js.configs.recommended,          // JavaScriptの推奨ルール
  ...tseslint.configs.recommended, // TypeScript向けの推奨ルール
  prettierConfig,                  // Prettierと競合するルールを無効化
  {
    rules: {
      '@typescript-eslint/no-explicit-any': 'error', // any禁止
    },
  },
)
```

3つのルールセットを重ねて適用しています。

| レイヤー | 内容 |
|---------|------|
| `js.configs.recommended` | `console.log` 残しや未定義変数など、JSの基本的なミスを検出 |
| `tseslint.configs.recommended` | TypeScript固有のミス（型の誤用など）を検出 |
| `prettierConfig` | PrettierとESLintのルールが競合しないよう、整形系のルールをOFFにする |

プロジェクト固有の追加ルール：
- `@typescript-eslint/no-explicit-any: 'error'` → `any` 型の使用をエラー扱いにする（警告ではなくビルド失敗レベル）

---

### `.prettierrc` — コード整形の設定

| 設定 | 値 | 効果 |
|------|----|------|
| `semi` | `false` | 文末のセミコロンをつけない |
| `singleQuote` | `true` | 文字列は `"` ではなく `'` を使う |
| `tabWidth` | `2` | インデントはスペース2個 |
| `trailingComma` | `"all"` | 配列・オブジェクトの最後の要素にもカンマをつける |
| `printWidth` | `100` | 1行100文字を超えたら折り返す |

`npm run format` を実行すると、このルールに従って `src/` 以下のコードが自動整形されます。

---

### `index.html` — HTMLの入口

| 記述 | 意味 |
|------|------|
| `lang="ja"` | スクリーンリーダーや検索エンジンに「日本語ページ」と伝える |
| `meta viewport` | スマートフォンで正しくレイアウトされるようにする |
| `<div id="app">` | TypeScriptがDOMを挿入する起点。中身は空でよい |
| `type="module"` | `import/export` 構文を使うESモジュールとして読み込む。これがないと `main.ts` 内の `import` 文が動作しない |
| `src="/src/main.ts"` | ViteがこのパスのTSファイルを開発時にそのまま処理する（本番ビルド時は自動でJSに変換） |

---

### 設定ファイル間の関係

```
tsconfig.json
    └─ 型チェックのルールを定義
    └─ noEmit: true なのでJSは出力しない

vite.config.ts
    └─ 実際のビルド・開発サーバーを担当
    └─ test.environment で vitest の動作環境も設定

eslint.config.ts
    └─ コードの書き方ルールを検査
    └─ prettierConfig を含むことで Prettier と競合しない

.prettierrc
    └─ 自動整形のスタイルルール
```

## コマンド

| コマンド | 内容 |
|----------|------|
| `npm run dev` | 開発サーバー起動 |
| `npm run build` | 本番用ファイルを `dist/` に生成 |
| `npm run test` | テストを1回実行 |
| `npm run test:watch` | ファイル保存のたびにテストを自動実行 |
| `npm run lint` | コード品質チェック |
| `npm run type-check` | TypeScriptの型エラーチェック |
| `npm run format` | コードを自動整形 |
