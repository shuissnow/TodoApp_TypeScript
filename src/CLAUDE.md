# src/ フォルダ — Claude 向けガイド

## 作業対象の判断

| 作業内容 | 対象フォルダ |
|---------|-------------|
| UI・画面・TypeScript コード | `src/frontend/` |
| API・DB・C# コード | `src/backend/` |

各フォルダの詳細なルールは、それぞれの `CLAUDE.md` を参照すること。

- フロントエンド: [`frontend/CLAUDE.md`](frontend/CLAUDE.md)
- バックエンド: [`backend/CLAUDE.md`](backend/CLAUDE.md)

---

## 共通ルール

- フロントエンドとバックエンドの変更を1つの作業で同時に行わない
- API 仕様の変更はバックエンドを先に実装し、フロントエンドを後から合わせる

---

## 作業単位ドキュメント

### 概要

特定の開発作業における「**今回何をするか**」を定義する一時的なドキュメント。
フロントエンド・バックエンドに応じて `frontend/` または `backend/` 配下に配置します。
作業完了後は参照用として保持されますが、新しい作業では新しいディレクトリを作成します。

### ディレクトリ命名

titleは作業単位の概要を簡潔にまとめたものにすること。

```
docs/04-implementation/[frontend|backend]/[YYYYMMDD]-[kebab-case-title]/
```

例：
- `docs/04-implementation/frontend/20260322-initial-implementation/`
- `docs/04-implementation/backend/20260328-add-backend/`

### 作成するファイルと順序

以下の順番で作成してください。
各ファイルを作成後に内容に重複や矛盾がないかを検討してください。
検討の上、問題がない場合に**1 ファイルごとにユーザーの確認・承認を得てから次に進むこと**。

1. `requirements.md` — 今回の作業の要求内容
    - 変更・追加する機能の説明
    - ユーザーストーリー
    - 受け入れ条件
    - 制約事項

2. `design.md`       — 実装アプローチと変更内容
    - 実装アプローチ
    - 変更するコンポーネント
    - データ構造の変更
    - 影響範囲の分析

3. `tasklist.md`     — 具体的な実装タスク
    - 具体的な実装タスク
    - タスクの進捗状況
    - 完了条件

4. `decision.md`     — （オプション）重要な決定事項の記録

### 禁止事項

- **過去の作業単位フォルダを修正しない。** 完了した作業の `requirements.md` / `design.md` / `tasklist.md` は意思決定の履歴記録であり、変更不可。
- **実装作業のたびに永続的ドキュメントを更新しない。** 01〜03 は設計変更時のみ更新する。
- **承認なしに複数ファイルを連続作成しない。** 必ず 1 ファイルごとに確認を取る。
